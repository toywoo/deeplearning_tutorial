# Lecture 12: Self-Supervised Learning (자기 지도 학습)

본 문서는 사용자의 강의 메모를 바탕으로 오탈자를 수정하고, 영어/한국어가 혼재된 개념을 매끄럽게 번역하여 `Core Question -> Key Mechanism -> Mathematical Foundation` 형식으로 구조화한 문서입니다. 새롭게 추가해주신 대조 학습(Contrastive)의 수식과 MoCo 프레임워크 등에 관한 설명을 심층 반영하였습니다.

자기 지도 학습은 데이터에 비싼 정답(레이블)을 달지 않고 원시 데이터 자체의 구조 안에서 유용한 특징을 강제로 학습하게 만드는 비전 및 언어 모델(LLM) 딥러닝 패러다임의 핵심입니다.

---

## 1. 사전 작업(Pretext)과 전이 학습의 패러다임
- **Core Question**: 세상에 넘쳐나는 엄청난 양의 "레이블(정답) 없는" 이미지 데이터 수백억 장을 인공지능이 스스로 소화하게 만들 방법이 있을까?
- **Key Mechanism**:
  - **사전 작업 (Pretext Task)**: 레이블이 없는 데이터셋의 일부 정보에 변형이나 가림을 가한 뒤, 신경망에게 원래 구성을 예측하도록 자체적인 '가짜 퍼즐 놀이'를 시킵니다. 인코더는 이 문제를 풀기 위해 이미지의 문맥과 유의미한 상식(Visual common sense)을 강제로 습득하게 됩니다.
  - **다운스트림 작업 (Downstream Task)**: 사전 작업 문제를 푸는 것 자체가 목적이 아닙니다. 여기서 익힌 범용적 의미 공간(Semantic Representation)의 인코더 가중치만 떼어내어, 분류기나 탐지기 같은 목표 작업(Downstream)에 미세 조정(Fine-tuning)을 하는 것이 메인 목표입니다.

## 2. 학습된 표현(Representation) 평가 프로토콜
레이블 없이 스스로 추출한 특징망(인코더)이 정말 똑똑한지 평가하는 기준 지표들입니다.
- **선형 평가 프로토콜 (Linear Evaluation Protocol)**: 사전 학습이 완료된 강력한 인코더의 가중치를 고정(Freeze)시킵니다. 그리고 맨 뒤에 선형 분류기 하나(Linear Layer)만 새로 달아서 성능을 봅니다. 이 얕은 한 층만으로도 정확도가 높다면 인코더가 이미 데이터를 선형적으로 잘 분리해 놓았다는 방증이 됩니다.
- **t-SNE & 클러스터링(Clustering)**: 고차원의 특징 임베딩을 시각화(2D, 3D 보기를 위한 차원 축소)하여, 모델이 스스로 비슷한 사물끼리 잘 뭉쳐놓았는지(Clustering) 살펴봅니다. 
- **견고성과 다운스트림 성능**: 특징들이 다른 데이터셋이나 변형에도 잘 일반화(Generalization)되는지, 최종 전이 학습 성능이 좋은지를 종합적으로 판단합니다.

## 3. 휴리스틱 기반 이미지 변형 사전 학습 (Image Transformations)
- **Core Question**: 초기 자가지도 학습 연구자들은 모델에게 어떤 종류의 '퍼즐'을 내야 이미지의 전반적 상식을 배울 수 있다고 보았을까?
- **Key Mechanism**:
  - **회전/수리/색칠 예측**: 이미지를 0, 90, 180도 등 무작위로 회전시키고 원래 각도를 맞히도록 하거나(Rotation), $3 \times 3$ 격자로 섞은 부분 이미지의 원래 자리를 맞히게 하거나(Jigsaw Rearrangement), 중간을 뚫어놓고 알아서 색을 칠하도록(Inpainting/Coloring) 훈련시킵니다.
  - **한계점**: 사람이 일일이 고안해낸 단일 사전 작업들은 매우 귀찮은 과정일 뿐아니라, 특정 작업(예: 퍼즐 맞추기)에만 편중된 특징만 학습되어 전체 다운스트림 태스크에 균일하게 범용적으로 쓰이기엔 어려움이 있었습니다.

## 4. 마스크 복원 기반 학습: MAE (Masked Autoencoders)
- **Core Question**: 픽셀을 순수하게 "물리적으로 압축"해 내는 방향으로 ViT(Vision Transformer)를 학습시키려면 어떻게 해야 할까?
- **Key Mechanism**:
  - 오토인코더의 구조의 혁신으로, ViT 구조를 기반으로 입력 패치의 무려 75%를 가려버립니다(Masking). 
  - **인코더(고도 압축)**: 매우 큰 트랜스포머 인코더에는 가려지지 않고 "살아남은 생존 패치(25%)"만 투입합니다. 
  - **디코더(데이터 증강 및 복원)**: 인코더의 압축 토큰과 빈 마스크 토큰들을 합쳐서 얕은 디코더에 통과시켜 원본 모자이크를 복원합니다. 학습 속도가 기하급수적으로 빨라져 이 방식은 ViT 자기 지도 학습의 표준 중 하나가 되었습니다.

## 5. 대조적 표현 학습 (Contrastive Representation Learning)
- **Core Question**: 앞선 마스킹이나 변환 사전 작업들은 모두 '원본 사진 한 장' 안에서의 변형일 뿐이었습니다. 데이터셋 내의 수백만 장의 '서로 다른 사진들'을 교차 시켜 "비슷한 놈은 가깝게, 다른 놈은 멀게" 묶어내는 고도로 추상적인 특징만 배울 수는 없을까?
- **Key Mechanism**:
  - **직관과 설계 (Intuition & Formulation)**: 
    - 사진 하나에 심한 증강(Augmentation: 자르기, 색상 변조 등)을 두 번 적용하면 각기 모습은 달라도 본질은 똑같은 2장의 쌍(**Positive Pair, $x^+$**)이 됩니다. 나머지 데이터셋의 수많은 무작위 사진들은 **Negative Pair ($x^-$)** 입니다.
    - 코사인 유사도를 이용해 기준 $x$와 식구인 $x^+$ 간의 점수(Score)는 극대화시키고, 나머지 남남인 무수한 $x^-$들과의 거리는 극단적으로 강하게 밀어내도록 모델을 밀어부칩니다.
- **Mathematical Foundation**:
  - 소프트맥스 기반 다중 분류기의 교차 엔트로피 손실함수(InfoNCE Loss) 구조를 취합니다.
  - $L = -\mathbb{E}_X \left[ \log \frac{\exp(\text{score}(f(x), f(x^+)))}{\exp(\text{score}(f(x), f(x^+))) + \sum_{j=1}^{N-1} \exp(\text{score}(f(x), f(x_j^-)))} \right]$
  - (분자는 Positive 쌍의 매칭 점수이고 분모는 Positive 쌍을 비롯해 비교 대상이 된 모든 Negative 쌍의 매칭 점수들의 총합입니다. 수많은 $N-1$개의 오답(Negative) 대비 정답(Positive)의 비중만 극대화 시키면 오차가 0에 수렴합니다.)

### 5.1 인스턴스 단일 대조 프레임워크의 진화 (Instance Contrastive)
- **SimCLR**: 단순하면서도 강력합니다. 오차 전파를 위해 막대한 Negative를 계산해야 하는 수식의 구조 상, 모델 훈련 시 배치(Batch Size)를 상상을 초월할 정도로 거대하게 세팅해야 성능이 올라가는 '대규모 하드웨어 자원 한계'의 단점이 있었습니다.
- **MoCo (Momentum Contrast)**: SimCLR의 이런 비효율적 학습 문제를 타파하기 위해 등장했습니다. 매번 큰 배치를 돌리지 않고, 수많은 과거 Negative 쌍의 벡터들을 큐(Queue 구조의 Memory Bank)에 한가득 모아두고 계속 빼서 쓰는 방식을 고안하여 자원 부하 문제를 해결했습니다. 

### 5.2 기타 파생 기술
- **CPC (Sequence contrastive learning)**: 연속적인 영상이나 오디오 시퀀스에서 다음을 예측할 때 대조학습 기법을 적용합니다.
- **DINO (Self-Distillation)**: Negative 쌍 없이도 선생-제자 구조망의 자체 증류를 통해, 대조 학습 없이 유사도를 높여 훌륭한 시각 분류 특징을 찾아내는 최신 방식 중 하나입니다.

## 🚀 추가 학습 및 실습 제안 (Further Explorations)

분산 훈련은 단순히 AI 모델링을 넘어서 컴퓨터 시스템 아키텍처에 대한 이해도를 요구합니다.

1. **[수학과 알고리즘 분석] 대조 손실 함수 (InfoNCE Loss)**:
   - 교차 엔트로피와 매우 닮아 있는 추가해주신 손실 함수의 공식을 다시 파보길 적극 권장합니다. 특히, 훈련 과정 중에 손실을 줄이려면 소프트맥스의 분모에 있는 네거티브의 수합($\sum_{j=1}^{N-1}$)을 지속적으로 눌러 코사인 유사도를 낮춰야 하는데 이것이 그래디언트 상에서 어떻게 작동하는지 수학적으로 탐구해 보세요.
2. **[간단한 코딩 실습] 직소 퍼즐 Pretext Task**:
   - `DL_scratch/source` 기반 환경이나 OpenCV/PyTorch를 사용하여 이미지 패치를 $3 \times 3$으로 커스터마이징해 섞고 9개의 레이블을 부여하여 ResNet 계열 분류기로 클래스를 잘 맞히는지 스크래치 테스트를 진행해 보세요.
3. **[논문 리딩] MoCo 개념 (선택형 심화)**:
   - *Momentum Contrast for Unsupervised Visual Representation Learning (He et al., 2020)*: SimCLR의 한계였던 커다란 모델 크기 배치를 뚫어버리고 '메모리 뱅크'와 '모멘텀 인코더'라는 혁신적인 구조로 메모리 최적화를 이뤄낸 MoCo 논문의 핵심 아키텍처 부분만 가볍게 읽어보는 것을 추천합니다.
