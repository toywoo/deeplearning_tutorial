## 1. 생성 모델의 기본 개념 (Generative Models)
- **Core Question**: 지도 학습이나 비지도 학습을 넘어, 모델이 데이터의 '구조'를 완벽히 파악하여 전에 없던 새로운 데이터(이미지, 텍스트)를 스스로 만들어낼 수 있을까?
- **Key Mechanism**:
  - **지도/비지도 학습의 차이**:
    - **지도 학습 (Supervised)**: 데이터 $x$와 정답 레이블 $y$의 쌍을 통해 $x$에서 $y$로 매핑하는 함수를 학습합니다 (분류, 회귀 등).
    - **비지도 학습 (Unsupervised)**: 정답 레이블 없이 데이터 $x$만 주어질 때 기저에 있는 구조적 특징을 찾아내는 것입니다. 목표 자체가 뚜렷하지 않으며 K-Means, 차원 축소(Dimensionality Reduction), 밀도 추정(Density Estimation) 등에 사용됩니다.
  - **생성형 모델 (Generative Models)**: 데이터 집합 $x$에 대한 확률 분포 자체를 학습하여, 존재하는 모든 샘플들이 확률 질량(연속일 경우 확률 밀도)을 놓고 경쟁하는 구도를 만듭니다. 이렇게 학습된 분포로부터 무작위 샘플링을 수행하면 기존에 없던 새로운 특성의 데이터를 모델이 생성해 낼 수 있게 됩니다.
  - **판별 모델 (Discriminative Models)과의 차이**: 판별 모델은 데이터가 주어졌을 때 정답의 확률 $P(y|x)$를 학습합니다. 하지만 이는 비합리적이거나 이상한 입력 값을 거부할 실질적 방법이 없습니다. 반면 생성 모델은 데이터가 어떤 형태의 분포를 띠는지 자체를 학습하기 때문에 이상치 감지(Anomaly Detection)가 잘 동작합니다.
- **개념 보충 (밀도와 분포의 관계)**:
  - 분포(Distribution)는 특정 데이터 값들이 나타날 가능성을 전체적으로 묘사하는 넓은 틀입니다. 반면 밀도(Density)는 연속적인 데이터(이미지 픽셀 등)에서 특정 구간에 데이터가 얼마나 집중적으로 존재할 확률이 높은지(빽빽함)를 보여주는 '분포 내에서의 확률 쏠림 정도'를 뜻합니다. 명시적 모델에게 있어서 밀도 함수를 훈련한다는 것은 실존할 만한 그럴싸한 이미지 쪽의 밀도 계수를 높여 모델이 더 그럴듯한 공간 내에서 샘플들을 상상하고 조합하도록 유도하는 행위입니다.

## 2. 생성 모델의 분류 체계 (Taxonomy)
생성 모델은 '명시적 밀도'를 직접 다룰 수 있는지 여부에 따라 크게 두 가지 갈래로 나뉩니다.
- **명시적 밀도 (Explicit Density)**: 
  - 확률(Density)분포를 수식 모델로 정의하고 계산하려 시도합니다.
  - **계산 가능 (Tractable)**: 정밀하게 밀도 값이 도출되지만, 픽셀 하나하나를 확률적으로 순차 계산하는 Autoregressive 방식이 주를 이뤄 연산 비용과 속도에서 매우 비효율적입니다.
  - **근사 계산 (Approximate)**: 완벽한 밀도 계산이 어려워 하한선 같은 근사치를 내어 최적화하는 방식을 취하며 가장 대표적으로 **VAE**가 있습니다.
- **암시적 밀도 (Implicit Density)**:
  - 확률 밀도 자체를 수치상으로 명시하려 하지 않고 그저 **"확률 분포에서 그럴싸하게 샘플(생성 결과)만 뽑아낼 수 있으면 된 것"**이라는 실용적 관점을 취합니다.
  - **Direct**: 신경망을 통해 잡음에서 다이렉트로 이미지를 샘플링하는 **GAN**.
  - **Indirect**: 반복적인 단계를 차근차근 밟으며 근사적으로 노이즈를 지워가는 간접 방식인 **Diffusion Models**.

## 3. Variational Autoencoders (VAEs)
- **Core Question**: 비지도 학습인 기존의 오토인코더를 개조하여, 압축된 잠재 공간(Latent Space)에서 단일 픽스된 벡터가 아닌 새로운 파동(확률적 분포)을 형성하여 다채로운 샘플들을 생성해내려면 어떻게 해야할까?
- **Key Mechanism**:
  - **기존 AE와의 차별점**: 기존 오토인코더(AEs)는 병목 현상을 유도해 원본 입력을 재구성하는 방식이라 그 본질은 단지 비지도 방식의 '압축'일 뿐 데이터를 무작위로 생성하지 못합니다. (잠재 공간이 연속적이지 않고 끊겨있어서 빈공간 추출 시 쓰레기 출력)
  - **VAE의 접근 및 Disentangling**: 이 단점을 극복하기 위해 VAE는 변수에 확률적 흔들림(Noise)을 가미합니다. 인코더가 $z$를 그냥 한 점이 아닌, 정규 분포를 이루는 평균($\mu$)과 분산($\sigma^2$)의 형태로 공간 상에 예측합니다. 이를 통해 추출된 임베딩(잠재 벡터)은 의미적으로 얽혀있지 않은 깔끔한 특징별 축(Disentanglement)을 형성하게 되어 모델링 및 생성을 다채롭게 유도합니다.
- **Mathematical Foundation**:
  - **베이즈 정리 및 근사가 필요한 이유 (`사용자 난해 지점`)**: 
    이상적인 성과를 내려면 데이터 발생 가능도(Likelihood)인 역확률 밀도 $P(x)$를 최대화하는 방식(MLE)을 쓰는 것이 맞지만, 실제 심층 모델 기반 조건부 확률 적분이 미분 불가능(intractable)할 정도로 수학적 연산이 심하게 막대해지는 현실 상황에 부딪힙니다. 즉, 베이즈 정리를 써서 전개하고 싶어도 데이터 분포의 절대 분모를 구할 수 없는 상황에 빠진 것입니다.
  - **해결책 (ELBO 최적화)**: 
    따라서 실제 미적분 전개가 불가능한 진짜 사후 확률 분포 $P(z|x)$를 구하는 것을 우회하여 가장 유사한 신경망 분포 예측망 $q_\phi(z|x)$를 활용하고, 예측 분포와 실제의 오차 간격(KL Divergence)을 패널티 주듯 축소해버리는 전략 수식을 펼칩니다. 
    이 전개 과정에서 모델 최적화의 증거인 **확률의 최하 보장선 Evidence Lower Bound (ELBO)** 가 도출되며, 이 하한선을 빡세게 높이기만 해도 간접적으로 우도(밀도)가 같이 높아져 모델이 성장해지는 정성적 증거를 얻게 됩니다.
  - $\log p_\theta(x) \ge \mathbb{E}_{z \sim q_\phi(z|x)} [\log p_\theta(x|z)] - \text{KL}(q_\phi(z|x) \| p(z))$

## 4. 생성적 적대 신경망 (GANs; Generative Adversarial Networks)
- **Core Question**: 골치 아픈 명시적 확률 연산이나 밀도 계산을 아예 우회하여, 단순한 신경망들의 적대적 피드백 경쟁만으로 가짜 데이터의 퀄리티를 최강으로 높일 수는 없을까?
- **Key Mechanism**:
  - **적대적 구조**: 판별자(Discriminator $D$)와 생성자(Generator $G$)라는 두 네트워크 간의 끊임없는 게임 이론(Minimax game)입니다. 초반 모델은 단순 우선(Prior) 분포(Gaussian 등) $p(z)$에서 무작위 확률 $z$를 가져와 생성기에 통과시킵니다 ($x = G(z)$). 판별기 $D$는 진짜 데이터베이스에 있던 진짜(Real=1)들과 $G$가 복제한 가짜(Fake=0)를 판독합니다.
  - **발전(DC-GAN 등)**: 이 구조는 단순 다층 퍼셉트론(MLP)에만 국한되지 않고 이미지 전용 계층 구조인 합성곱 신경망을 적극 차용한 **DC-GAN(2015)** 시대를 기점으로 크게 실증적 성과를 만들어냈습니다. 고해상의 침실 이미지를 만들고, Latent Space Interpolation, Style GAN 등의 수많은 변형체를 낳으며 대생성 시대의 포문을 열었습니다. (이 GAN의 창시자인 이안 굿펠로와 그 랩실 구성원은 훗날 LLM인 GPT를 세상에 내놓기도 합니다.)
- **Mathematical Foundation**:
  - $\min_G \max_D \left( \mathbb{E}_{x \sim P_{data}} [\log D(x)] + \mathbb{E}_{z \sim P_z} [\log (1 - D(G(z)))] \right)$
  - **초기 그래디언트 소실 해법**: 
    학습 초반엔 생성자($G$)가 만드는 가짜의 퀄리티가 너무 형편없어서, 분류기능을 수행하는 판별자($D$)가 조금만 봐도 무조건 쉽게 적발해냅니다 ($D(G(z)) \approx 0$). 수학적 구조상 이렇게 한쪽이 너무 빨리 우세해버리면 오차 전파를 돕는 도함수(Gradient)가 소멸되어 생성자의 실력이 영원히 성숙하지 못하는 문제가 터집니다.
  - **Solution**: 실무에서는 $G$ 모델을 위한 우측 수식 중 $\log(1 - D(G(z)))$를 단순히 최소화하는 방식을 버리고, 오히려 $-\log(D(G(z)))$를 최대화하도록 고의적으로 공식을 바꾸어, 적발당하는 초기 구간일지라도 생성기의 피드백 기울기가 크게 발생하도록 유도해 학습력을 강제 견인시킵니다.
- **GAN의 특성 요약**:
  - **Pros**: 심플한 구조에서 매우 뛰어난 화질 퀄리티를 지닌 단일 샘플들을 생성.
  - **Cons**: 두 이질적망의 훈련 불안정성, 수렴을 확인할 직관적인 Loss 커브 부재, 초거대 규모 확장이 어려움.

## 5. 확산 모델 (Diffusion Models)
- **Core Question**: 생성계의 전성기를 견인하던 GAN의 아성(훈련 불안정, 구조적 한계)을 타파하고, 샘플 궤적 상의 물리학적 잡음을 서서히 걷어내는 것만으로 지독할 정도로 선명하고 안정적인 초해상도 이미지 생성을 해낼 수는 없을까?
- **Key Mechanism**:
  - 현재 시대를 주름잡는 영상 생성계의 최강자입니다 (GAN을 완전히 밀어냈습니다.)
  - 근본 원리는 입력 이미지를 확률적 노이즈가 있는 단계(0은 클린, 1은 가우시안 완전 노이즈)로 천천히 파괴하고 다시 노이즈를 덜어내는(Denoise) 과정을 모델의 신경망에 반복 학습시켜 간접적으로 원본 이미지 매핑 궤적을 잇는 구조입니다. (수학적으로 매우 난해하며 여기선 아주 개괄적인 정류 흐름 모델 등에 국한합니다.)
  - **Rectified Flow (노이즈 선형보간)**: 데이터 분포($x$)와 노이즈 분포($z$) 사이를 잇는 어떤 파도 타기 궤적인 수치화 벡터 $t$ 선을 놓고 $t$율 만큼 섞은 상태 구조를 훈련시켜 점진 역추적하는 오늘날 현대적 일반 구성기법입니다.
- **Mathematical Foundation & Code Explanation** (`사용자 요청 보강해설`):
  
  **Training (학습 과정)**:
  데이터에 의도적인 소음을 심어주고 모델에게 그 소음의 정체를 파악해 내라고 지시하는 구조입니다.
  ```python
  for (x, y) in dataset:                  # 원본 이미지(x)와 라벨 텍스트(y) 로드
      z = torch.randn_like(x)             # x와 형태, 해상도가 완벽히 동일한 가우시안 잡음을 생성
      t = random.uniform(0, 1)            # 현재 시나리오에서 노이즈화할 심각성 정도 (1에 가까울수록 심한 노이즈 사진)
      
      xt = (1 - t) * x + t * z            # 이게 Rectified flow 보간: 비율(t)만큼 뒤섞어 고의 망친 사진들(xt) 형성
      
      # if random.random() < 0.5: y = y_null  <-- (CFG에 나오는 마법: 조건 정보 y를 일부러 반 날림)
      
      v = model(xt, y, t)                 # 오염 수준이 된 망친 사진(xt)과 오염 비율(t)을 넣으면 모델 신경망이 숨어있는 노이즈(v)를 예측함 
      loss = (z - x - v).square().sum()   # [모델 예측값(v) VS 정체불명오차(z-x-v)] 간의 순수 간격 손실(MSE Loss) 최적화로 학습 진행
  ```

  **Sampling (추론 과정)**:
  지직거리는 도화지에 모델(지우개)을 연달아 찍어대며 형태의 본질을 만들어 버립니다.
  ```python
  y = user_input()                        # 사용자의 프롬프트 조건 명령 입력 (예: 선명한 호랑이를 부탁해) 
  sample = torch.randn(x_shape)           # 맨 처음엔 무조건 그냥 100% 임의의 수학적 노이즈 덩어리(빈 도화지)에서 시작
  
  for t in torch.linspace(1, 0, num_steps): # t를 심한 상태 1에서 원본인 0으로 조금씩 좁혀나가는 반복 포문 시동
      v = model(sample, y, t)             # 빈 도화지와 요구 조건(y)을 넣으면 모델이 '지워야할 오보적 노이즈 방향성(v)'을 응답
      sample = sample - v / num_steps     # 잔류 노이즈 그림에서 지워야할 오차를 뺌으로써(Denoise) 서서히 원하던 이미지가 환상처럼 피어남
  ```

### 5.1 Classifier-Free Guidance (CFG)
- **Core Question**: 우리가 입력한 프롬프트(조건, $y$)를 확산 신경망이 훨씬 더 고집스럽고 뚜렷하게(Guidance) 반영하여 그려낼 수 있도록 하려면, 수학적으로 어떤 강제 수치를 부여해 줘야 할까? (`사용자 요청 보강해설`)
- **Key Mechanism**:
  - 만약 일반 조건적 모델 방식만 쓰게 되면, 이미지는 모델이 학습 과정에서 가장 흔하게 봐왔던 쪽으로 적당히 피신해버려 프롬프트 내용 지시력(Conditioning)을 곧잘 무시해버리는 품질 약화 문제가 발생합니다.
  - 위 파이썬 훈련 코드 단의 주석된 부분을 활성화 하면 데이터의 절반은 라벨 조건을 비우고(`y_null`) 훈련을 병행하게 됩니다. 이것은 하나의 거대 다중 모델에서 "조건이 달린 방향성 확률" 과 "분류 조건 없는 기초 방향성 확률"의 기술을 완전히 동시에 학습시키는 것입니다.
  - **수학적 응용**:
    $V_{cfg} = (1+w) V_{cond[y]} - w V_{uncond[\emptyset]}$ 
    CFG는 이 수식을 통해 놀라운 증폭 성능을 보장받습니다. 프롬프트 지시를 받았을 때(조건 $v_o$) 예측한 부분과 아무렇게나 생성해낼 방향(비조건 $v_y$) 예측 부분 사이의 **상대적 격차(Points difference)** 를 구합니다. 
    그리고 이 격차(순구하게 사용자의 텍스트 방향성만이 가지는 궤도)에 커다란 가중치 스케일링($w$)을 과도할 만큼 듬뿍 곱해 모델에 무력으로 욱여넣는 것입니다. 이것이 지도가 없는 생성(Free)에서 분류기의 지향성(Guidance)만 압도적으로 뽑아내는 놀라운 트릭이자 기술입니다.

### 5.2 노이즈 스케줄링과 고해상도 생성 (Advanced Diffusion Techniques)
- **Optimal Prediction과 노이즈 스케줄링 (Noise Schedules)**
  - 확산 모델 훈련 과정에서 중간 수준의 노이즈($t \approx 0.5$) 구간은 원본의 형체를 파악하기도, 그냥 노이즈로 보기도 애매하여 모델이 예측하기 가장 어려워하는(Ambiguous) 구간입니다. 반면 퓨어 노이즈($t=1$)나 클린 이미지($t=0$) 구간은 최적화 예측이 상대적으로 쉽습니다.
  - 기존 훈련 시 모든 노이즈 스텝($t$)에 동일한 가중치를 부여하는 문제를 극복하기 위해, **Logit-normal sampling** 기법 등을 활용하여 모델이 가장 헷갈려하는 중간 노이즈 대역에 더 많은 학습 가중치를 배치하는 등 스케줄링 조절 기법을 씁니다. (고해상도로 갈수록 픽셀 간 상관관계를 고려해 평균 노이즈 배율을 높게 시프트시키는 형태를 취합니다.)

- **잠재 확산 모델 (LDM; Latent Diffusion Models)**
  - 확산 모델을 원본 이미지의 '무거운 픽셀 공간 전체'에 직접 돌리는 것은 고해상도(4K 등) 확장 시 지독한 연산량 한계를 가져옵니다.
  - 이를 막기 위해 VAE(오토인코더) 구조를 결합합니다. **(1)** 인코더로 원본을 저차원(압축된) 잠재 공간(Latent Space)으로 축소합니다. **(2)** 압축된 작은 잠재 벡터 위에서 확산 모델을 여러 번 반복해 노이즈를 지우고 시그널(데이터 본질)을 뽑아냅니다. **(3)** 최종 결과 벡터를 VAE 디코더로 업샘플링(복원) 해 고해상도 이미지를 얻습니다.
  - 단, VAE 디코더 복원 시 특유의 흐릿한(Blurry) 질감이 나타날 수 있어 특별히 디코더 결단부에 GAN 손실(적대적 모델 평가)을 연결하여 경계면을 매끄럽고 선명하게 깎아내는 완성형 파이프라인을 구축합니다.

- **확산 트랜스포머 (DiT; Diffusion Transformers)**
  - 영상을 복원하는 기존 확산 모델의 코어망(U-Net 구조)을 최근 추세인 Transformer 블록으로 교체하는 기법입니다. 
  - **조건 주입 (Conditioning in DiT):** 표준 트랜스포머 내부로 텍스트 같은 조건식 정보($y$)를 어떻게 주입(Injecting)할까요? 확산 블록 중간 중간에 있는 정규화 계층인 AdaLN(Adaptive LayerNorm) 등을 사용하여, 조건식 $y$의 정보가 피처(Feature) 활성화 값의 **스케일(Scale)과 시프트(Shift)**를 예측 및 조절하도록 교묘히 설계하여 토큰 시퀀스로 합류시킵니다. 

- **비디오 및 텍스트 전환 생성 (Text to Image or Video)**
  - 전체 파이프라인은 `Text Encoder -> Diffusion Model -> Decoder` 로 구성됩니다. 
  - 특히 비디오 생성 시에는 프레임이 긴 시간적(Temporal) 시퀀스로 주어지므로 처리해야 할 토큰의 수가 압도적으로 폭증해 DiT와 3D 합성곱 등의 고도화 연산이 요구됩니다.

### 5.3 점수 함수 기반 모델과 가속화 (Score-based Models & Distillation)
- **일반화된 확산 모델 (Generalized Diffusion)**
  - 위에서 설명한 확산식들은 $x_{t} = a(t)x + b(t)z$ 형태의 더욱 일반적인 분산 제어 함수 폼으로 규합할 수 있습니다. 
  - 노이즈를 섞을 때 데이터 공간 확장의 분산(Variance) 변화 추세에 따라 최적화 기법이 달라지며, 이는 대표적으로 **분산 보존(VP; Variance Preserving)** 방식과 노이즈율을 계속 키워 스케일 분산을 폭발시키는 **분산 폭발(VE; Variance Exploding)** 방식으로 구조화 됩니다.
  
- **점수 함수 (Score Function) 및 미분 방정식 (SDE/ODE) 체계** (`사용자 요청 보강해설`)
  - 확산 모델 철학은 통계 물리학의 **점수 기반 생성 모델링(Score-based Generative Modeling, $\nabla_x \log p(x)$)** 개념과 완전히 일치됩니다. 확률 밀도가 더 높은 '진짜 이미지' 영역을 향해 데이터들이 움직이려는 등고선 기울기 백터를 '점수'라 부르며, 노이즈 잔량을 예측하는 확산 모델은 결국 이 '점수 함수'를 학습하는 것과 수학적으로 동일합니다.
  - 이를 응용해 시간 스텝($t$)을 100단계, 1000단계의 이산형이 아니라 매끄러운 연속 수치 변수로 무한히 쪼개 취급해버리면, 앞선 과정 전체가 노이즈를 교섭하는 거대한 **확률 미분 방정식(SDE)**이나 **상미분 방정식(ODE)** 수식 체계로 완전히 치환됩니다. 이 통찰력 덕에 현대 수학 모듈인 미분 수치해석 Solver 기술들을 코드에 그대로 이식할 수 있게 되었습니다.

- **확산 모델 증류 (Diffusion Distillation)** (`논문 보강 추가`)
  - 디퓨전 모델의 가장 무거운 치명적 단점인 수백 번의 재반복 추론 속도 지연(High Latency) 현상을 타파하여 빠르게 생성할 수 있도록 스텝을 건너 뛰는(Skipping) 가속화 압축 기술입니다.
  - 100번 스텝 정답을 내리는 커다란 확산망(선생)의 중간 과정을 무시하고 단 1번~4번의 추론 스텝만 껑충껑충 뛰어 최종 정답 근처로 예측해내도록 새로운 학생 모델을 재훈련(지식 증류)시킵니다. 스텝을 극단적으로 줄이면 화질(품질)이 저하된다는 트레이드 오프 한계가 있었습니다.

## 🚀 추가 학습 및 실습 제안 (Further Explorations)

분산된 디퓨전 연구들이 공학적으로 어떻게 결합되고 가속화되었는지를 살펴보는 방향의 실습을 제안합니다.

1. **[수학과 확률 추론] 변분 추론과 ELBO 증명 (Variational Inference)**:
   - 디퓨전 모델에서 VAE의 ELBO Loss가 어떻게 연관되는지, $P(x)$ 전개 과정을 통해 Kullback-Leibler (KL) Divergence 로스 구조가 분포에 어떤 패널티를 주는 지 손으로 직접 수식을 유도해 볼 것을 권장합니다.
2. **[논문 리딩] 확산 모델 가속화 및 증류 기술 (Distillation)** (`사용자 요청 보강`):
   - *Progressive Distillation for Fast Sampling of Diffusion Models (Salimans et al.)* 또는 *Consistency Models (Song et al.)*: 왜 물리적으로 디퓨전은 느린 추론 방식 태생 한계를 갖는지, 그리고 이것을 극복해서 반으로 스텝을 계속 접어 나가는 파이프라인 지식증류 논문을 훑어보는 방법이 관련 개념 확장에 강한 도움이 됩니다.
3. **[논문 리딩] 잠재 확산 모델 구조 (LDM)**:
   - *High-Resolution Image Synthesis with Latent Diffusion Models (Rombach et al.)*: 미드저니, 스테이블 디퓨전 등 모든 상용 고품질 엔진의 베이스 모델 포맷입니다. Pixel Level의 비효율성을 어떻게 VAE 구조와 UNet 코어로 뚫고 탈출했는지 그 아키텍처 스토리를 분석해 보는 것이 좋습니다.
