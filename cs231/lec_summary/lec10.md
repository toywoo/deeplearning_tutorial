# Lecture 10: Video Understanding (비디오 이해)

본 문서는 사용자의 기존 강의 노트를 바탕으로 오탈자를 수정하고, `Core Question -> Key Mechanism -> Mathematical Foundation` 형식의 구조화와 보충 설명을 더해 깊이 있는 문서로 재작성한 확장본(Expanded Version) 문서입니다.

비디오는 단순한 2D 공간(이미지)을 넘어 '시간(Time)' 차원이 결합된 데이터입니다. 모델 내부적으로는 프레임 축(T)이 추가된 4D Tensor ($N \times T \times C \times H \times W$, 여기서 $N$은 배치, $T$는 연속된 프레임 수)를 입력으로 다룹니다.

---

## 1. 비디오 분류 및 통신 전략 기초
- **Core Question**: 데이터 용량이 막대하고 해상도가 높은 비디오의 연속적인 영상 흐름을 딥러닝 네트워크에 어떻게 투입할 것인가?
- **Key Mechanism**:
  - **다운샘플링(Downsampling) 및 클리핑(Clipping)**: 수만 프레임을 한 번에 분석하기는 연산 및 메모리 제약상 불가능합니다. 따라서 프레임 속도와 화면 해상도를 낮추고, 2초~5초 분량의 짧은 무작위 프레임 덩어리(클립) 단위로 잘라 학습시키는 샘플링 전략을 사용합니다.
  - **다중 프레임 통합 (Fusion Strategies)**: 추출된 프레임들을 일반적인 이미지용 2D CNN 구조에서 어떻게 융합할지에 대한 고찰입니다.
    - **Late Fusion (후기 융합)**: 여러 프레임을 각각 독립적으로 처리하고, 거의 마지막 FC(전결합) 레이어나 풀링 단계에서 벡터들을 합치거나 예측 결과를 평균 냅니다. 고차원 정보를 합치지만 움직임 흐름을 심층적으로 추적하지 못합니다.
    - **Early Fusion (조기 융합)**: 첫 번째 컨볼루션 레이어 직전 입력 단계에서 여러 프레임 이미지를 채널(Channel) 깊이 방향으로 합쳐(Stack) 쌓은 입력 텐서를 만듭니다. 가장 빨리 시간 정보를 물리적으로 통합하지만 공간적/시간적 고수준 연관성을 갖기엔 부족합니다.

## 2. 시공간을 확장한 해법: 3D CNN
- **Core Question**: 비디오가 지닌 '물리적 외형(공간 차원)'과 '순차적 움직임 패턴(시간 차원)'을 동시에 포착하면서도 다룰 파라미터를 효율적으로 공유하려면 어떻게 해야 할까?
- **Key Mechanism**:
  - **3D 커널의 도입**: 2D 컨볼루션 필터(커널)가 좌우/상하로 움직였다면, 시간 축 깊이를 가진 3D 커널(시간 $\times$ 높이 $\times$ 너비)로 시간 축까지 입체적으로 슬라이딩합니다.
  - **장점**: 2D 컨볼루션의 공간적 변이 불변성(Spatial Translation Invariance, 예를 들어 고양이가 화면 좌상단에 뜨든 가운데 뜨든 고양이로 탐지) 기능이 시간 축 방향의 불변성(Temporal Shift Invariance)으로 확장되어, 어느 프레임(시간대)에 동작이 발생하든지 동일하게 좋은 특징을 추출합니다.
  - **C3D (The VGG of 3D CNNs)**: 초창기의 매우 직관적인 3D CNN 아키텍처입니다. VGGNet 16 계열의 레이어 구성에 시간(Z축) 차원을 단순 부가한 형태입니다. 그러나 부동소수점 연산량(GFLOPs)이 기본 VGG-16 대비 거의 3배로 증가하여 당시나 지금이나 계산 효율(비용) 측면에서 매우 무겁습니다. ResNet의 잔차 연결(Residual Block)을 더한 모델들도 이어서 개발되었습니다.

## 3. Two-Stream Networks와 광학 흐름 (Optical Flow)
- **Core Question**: 인간은 어둠 속이나 흑백 반전 상태에서도 윤곽의 움직임만으로 행동 유형을 파악할 수 있는데 인공지능도 동작과 외형을 분리하여 학습할 수 있을까?
- **Key Mechanism**:
  - 인간 두뇌가 외형 정보(Ventral stream) 영역과 공간 모션(Dorsal stream) 처리 영역을 따로 갖는다는 점을 착안, 네트워크 자체를 서로 다른 입력을 받는 두 스트림(Two-stream)으로 병렬로 나눕니다.
  - **Appearance Stream (외형 분류기)**: 단일 RGB 프레임 하나 만을 받고 2D CNN을 통과하여 정적인 모양 혹은 맥락적 단락을 판단합니다.
  - **Motion Stream (동작 분류기) 및 Optical Flow**: 이 스트림은 픽셀의 이동 방향 벡터인 광학 흐름 지도 여러 장을 받습니다. 조도의 항상성(동일 물체의 밝기는 움직여도 픽셀 단위에서 유지된다는 가정)에 기반해 이전 프레임 대비 다음 프레임에서의 이동 방향을 뽑아낸 물리적 정보 벡터입니다. 
  - 정적인 컬러 배경/질감을 보지 않고 순수 움직임에 집중하므로 과적합(Overfitting) 예방에 효과적이며 최종 파이프라인 성능을 개선했습니다. 

## 4. 장기적(Long-term) 시퀀스 모델링 및 I3D
- **Core Question**: 긴 클립 전체의 포괄적인 연관성을 학습하거나, 무거운 3D CNN의 가중치 초기 학습 과정을 빠르게 할 방법은?
- **Key Mechanism**:
  - **CNN + RNN 융합형**: 각 프레임 단위의 특징을 순차적으로 강력한 시간축 다루기 도구인 RNN(LSTM, GRU 등) 셀에 입력하는 방식입니다. 하지만 너무 긴 입력은 정보 소실과 기울기 소실 발생 가능성이 있고 순차 처리로 인한 병목 탓에 매우 느렸습니다.
  - **Non-local Neural Networks (Self-attention)**: 트랜스포머의 셀프 어텐션(Self-Attention) 개념을 차용해 현재 프레임 분석 시 거리가 먼 다른 여러 전체 프레임들과 내적 비교를 수행하여(Non-Local Block), 위치에 얽매이지 않고 장기적인 정보를 병렬 처리 추출합니다.
  - **I3D (Inflated 3D ConvNet)**: 매번 3D 필터를 밑바닥부터 대규모 비디오 데이터로 학습하기란 버겁습니다. 그래서 ImageNet으로 잘 훈련된 2D CNN 가중치를 가져와 단순히 시간 축으로 복제(부풀리기, Inflation)하여 3D 커널을 만드는데, 이 방식이 매우 높은 성능 이점을 가져다 주어 현재도 많이 변형 사용됩니다.

## 5. 시공간 객체 탐지 및 기타 기법 (Spatio-Temporal Detection & Multimodal)
- 지금껏 클립 전체의 단일 액션을 분류했다면 추가적으로 해당 액션을 취하는 목표 객체의 **레이블, 2D 바운딩 박스 제한, 더불어 액션의 시작과 끝점(Time window)** 을 통합 예측하는 Spatio-Temporal Detection(Faster R-CNN 구조를 Tube화) 연구가 병행되었습니다. 
- 비디오에는 기본적으로 오디오(소리) 스트림이 결합되어 있어, 서로 시청각 모듈의 레이블을 교차 보정해주거나 시각에 영향을 미치는 다중 모달 접근 모델들도 생겨났습니다.
- 극강의 프레임 샘플링 성능을 높이는 라이트웨이트 네트워크 구조(SCSampler, X3D, SlowFast Network) 등도 지속 발전하고 있습니다.

## 🚀 추가 학습 및 실습 제안 (Further Explorations)

비디오 처리는 기존 이미지보다 무겁기에 메모리 관리와 최적화가 필수적입니다. 다음 사항들을 깊이 연구해 보세요.

1. **[수학 및 알고리즘 구현] Optical Flow 메커니즘 전개 구현해보기**:
   - OpenCV 등에서 제공하는 Optical Flow 대신, 그 수학적 근간인 프레임간 시간 미분($I_t$) 전개 방정식이나 고전적인 알고리즘(Lucas-Kanade 방식)의 기본 구조를 이해하고 직접 NumPy 환경에서 그라디언트를 계산해 벡터 지도를 시각화 해보세요.
2. **[I3D 모델 힌트] 초기 가중치 부풀리기 시뮬레이션**:
   - I3D(`Inflation`)가 무엇인지 머리로만 이해하기보다, 기존 $3 \times 3$ NumPy 가중치 행렬이 있을 때 파이썬 코드로 시간축의 $1/T$ 만큼 스케일링하며 형태를 $3 \times 3 \times 3$ 커널로 복사하는 (Numpy repeat & reshape 활용) 스크립트를 작성해 동작을 검증해보세요.
3. **[학술 논문 접근 제안]**:
   - *Two-Stream Convolutional Networks for Action Recognition in Videos (NIPS 2014)*: 공간과 시간에 대한 두 스트림 분리를 공식적으로 제안한 컴퓨터 비전 역사상 중요한 문헌.
   - *Quo Vadis, Action Recognition? A New Model and the Kinetics Dataset (CVPR 2017)*: I3D 아키텍처와 함께 딥러닝 비디오의 기초 표준인 Kinetics 데이터셋을 소개한 논문입니다.
