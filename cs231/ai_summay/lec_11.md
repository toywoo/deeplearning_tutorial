# Lecture 11: Large Scale Distributed Training

## 1. 단계별 요약 (Step-by-step Summary)
- **Step 1: 데이터 병렬화 (Data Parallelism)**: 모델 크기는 단일 GPU에 들어가지만 데이터셋이 수억 장일 때, GPU 100대를 사서 각각의 GPU에 동일한 모델 복사본을 넣고 데이터만 1/100 씩 찢어서 학습시킨 뒤 기울기(Gradient) 결과만 인터넷 선으로 동기화 결합하는 가장 흔한 분산 훈련입니다.
- **Step 2: 모델 병렬화 (Model Parallelism)**: Llama 3나 GPT-4 처럼 파라미터가 1테라바이트가 넘어버리면 GPU 한대에 아예 모델 코드가 안올라갑니다. 따라서 네트워크 1층~10층은 GPU 0번에, 11층~20층은 GPU 1번에 박아두고, 순방향 역방향 시마다 통신 인프라 핑을 계속 때리며 이어달리기(파이프라인)를 하는 극한 기술.
- **Step 3: 통신 오버헤드 (All-Reduce 최적화)**: GPU 1만 개가 매 스텝마다 서로에게 파라미터를 브로드캐스트 전송하면 인터넷 회선이 폭파됩니다. Ring All-Reduce 구조처럼 옆 사람에게만 차례로 데이터를 넘기며 둥글게 원형 버퍼 링으로 그레디언트를 통일시키는 인프라 공학.
- **Step 4: ZeRO Optimizer 등 최적화**: GPU마다 최적화 메모리(Adam의 momentum)조차 나눠 가지게(Sharding) 하여 VRAM 한계까지 쥐어짜내는 마이크로소프트의 초대형 언어/비전 훈련 기술들의 핵심들을 조명합니다.

## 2. 전문 용어 해설
- **Pipeline Parallelism (파이프라인 병렬성)**: 모델 파이프라인 구간별로 GPU가 다르면, 앞 놈이 연산할 때 뒷 놈이 백수처럼 노는 '버블 현상'이 생깁니다. 이 유휴 공백시간에 다른 미니배치를 교차로 섞어 집어넣어서 GPU 100% 가동률을 뽑아먹는 교대 연산 전략입니다.
- **Ring All-Reduce (링 통신망 동기화)**: 수천 대의 그래픽 카드가 마스터 서버 하나에 모조리 신호를 송신하는 병목을 해소하기 위해 카드들을 "물레방앗간 원형 모양 띠" 로 결선하여 자기 우측 이웃 카드 딱 1명 한테만 조각 데이터를 보내 대역폭을 보장하는 방식입니다.

## 3. 맥락 보완
- 과거 스탠포드 강의에 없었으나 2025년 들어 이 모듈 1시간 분량이 아예 1대장 급으로 커버된 이유는 거대 덩치(LLM 급 1,000빌리언 이상 파라미터 ViT 혼합 모델) 분산 연산이 안되면 아무리 수학 알고리즘이 좋아도 테크 기업 구동이 원천 불가능해 이 분산 아키텍트 지식 없이는 딥러닝 취업이 안 되기 때문입니다.

## 4. 비유를 통한 설명
- **Data Parallelism (데이터 병렬화)**: 공장에 수만 장 우표 붙일 물량이 오면, 100명의 직원(GPU)에게 다 똑같은 우표 부착 가이드라인(복제 모델)을 주고 100장씩 나눠서 평행 작업 한뒤 취합만 하는 것입니다.
- **Model / Pipeline Parallelism (모델 파이프라인)**: 자동차 한 대 조각조가 엄청나게 커서, 1번 공장 직원은 휠만 조립해서 2번 직원 공장으로 넘기고 2번 직원은 유리만 끼워서 3번 공장으로 넘기는 일직선 릴레이 공정 라인. 대신 1번이 놀지 않게 계속 다음 차 휠을 돌려야 합니다.

## 5. 논문 소개
- **ZeRO: Memory Optimizations Toward Training Trillion Parameter Models (Rajbhandari 등 Microsoft, 2020)**: 모델의 State 파라미터를 여러 GPU에 지능적으로 쪼개어 수조 개급 LLM이나 멀티모달 모델들의 훈련 길잡이가 된 분산 처리 역작 시스템 논문.
