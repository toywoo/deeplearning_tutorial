Dropout
정규화에 일종으로 훈런시에는 무작위성을 추가(뉴런을 0으로 만든다)하고 테스트 시에는 제거하는 것이다. 모델이 훈련 데이터를 학습하기 어렵게 만드는 것이지만 모델의 일반화 성능이 향상된다. 하지만 이런 경우에는 테스트 시에 출력 값이 커지게 된다. 왜냐하면 뉴런이 다 켜지면서 활성화 값이 다 합해지기 때문에 그래서 스케일링을 해야 한다. 

Activation Function
비선형성을 추가하고 시그모이드 함수를 주로 사용하지만 요즘은 안 쓴다. 입력 값이 매우 크거나 작으면 여러 겹을 전파 될 수록 기울기가 점점 작아진다. 그래서 ReLU를 사용한다. 그리고 최댓값을 얻는 연산이 더 빠르다. 하지만 이 함수도 음수일 때 평평한 점이 존재한다. 그래서 GELU가 있다. 가까운 영역에서 활성화 함수의 기울기를 일정하게 유지하는 방식으로 이 문제를 해결하는 활성화 함수를 쓰고 있다.
활성화 함수는 선형 연산자 뒤에 사용한다.

CNN Architecture

VGGNet
작은 필터와 더 깊은 네트워크 AlexNet에 비해서 레이어를 늘렸다. 그런데 왜 3x3행렬로 합성곱을 할까? 3x3 행렬은 3번의 합성곱 연산을 거치면 7x7 필터와 같아진다. 이렇게 매개변수가 작아질수록 입력 데이터간의 더 복잡한 모델링이 가능하기 때문이다.

ResNet
레이어를 더 깊게 쌓으면 어떻게 될까? 20개의 레이어로 구성된 모델이 56개의 레이어로 구성된 모델보다 테스트 오류가 더 낮다는 것이다. 즉 성능이 좋다는 것이다. 이론적으로 얕은 신경망이 모델이 표현은 더 적게한다. 하지만 20층 모델에 항등함수로 36층을 추가하여 56층을 만들어서 학습 시키면 동일한 성능을 내야 하지만 동일한 성능이 아닌 20 개의 레이어 모델이 더 성능이 좋다. 그래서 모델의 용량인 층을 높이는거 때문이 아니라 복잡한 비선형 구조 때문에 최적의 해를 찾기 힘든 것이다. 
그럼 어떻게 더 깊은 모델의 성능을 앝은 모델보다 높일 수 있을까? 잔여(residual)학습을 하면 되는 것이다. 그냥 입력을 출력에 더하는 것이다. 그런 다음 활성화 함수에 넣는다. 그런 다음 풀링을 진행한다.
